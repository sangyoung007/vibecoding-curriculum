01 STEP 01 / 기초 파일 분석 & 초안 생성

엑셀 분석 및 HTML 프로그램 초안 생성

목표: 엑셀 분석 및 HTML 초안 생성 엑셀 업로드 전체 표 출력 통합 검색창
설명

이번 교육은 코딩 문법을 어렵게 배우는 과정이 아닙니다. 대신 인공지능(LLM)에게 똑똑하게 일을 시키는 파트너십과 협업 방식을 배우는 과정입니다. 준비된 예제 엑셀 파일을 통해 실제 업무 프로그램을 직접 만들어보는 경험을 시작할 것입니다. 지금 생성되는 첫 번째 HTML 파일이 앞으로 우리가 계속 살을 붙여나갈 훌륭한 시작점이 되니, 처음부터 완벽하게 만들려고 부담을 가지지 않으셔도 괜찮습니다.

교육생 실습 내용
  1. 예제 엑셀 파일 실습_표준_검색데이터.xlsx를 LLM에 첨부한다.
  2. 아래 프롬프트를 입력한다.
  3. 생성된 코드를 복사한다.
  4. 메모장에 붙여넣고 업무요청검색.html로 저장한다.
  5. 저장한 HTML 파일을 브라우저로 실행한다.
입력할 프롬프트
첨부한 엑셀 파일은 업무 요청 검색 프로그램 실습용 데이터이다. 먼저 엑셀의 시트, 컬럼, 데이터 특성을 분석한 뒤, 이 파일 구조에 맞는 HTML 프로그램을 만들어줘. 목표: - HTML 파일 하나만 생성한다. - 브라우저에서 바로 실행되어야 한다. - VSCode, Node.js, Python 설치 없이 동작해야 한다. - 사용자가 엑셀 파일을 업로드하면 데이터를 표로 보여준다. - SheetJS CDN을 사용해도 된다. - 엑셀의 첫 번째 시트를 읽는다. - 첫 번째 행은 컬럼명으로 사용한다. - 전체 데이터를 표로 보여준다. - 상단에 통합 검색창을 만든다. - 검색어는 전체 컬럼을 대상으로 검색한다. - 검색 결과 건수를 표시한다. 중요: - 컬럼명은 첨부 엑셀의 실제 컬럼명을 그대로 사용한다. - 코드는 하나의 HTML 파일으로 완성해줘. - 설명은 짧게 하고, 최종 HTML 코드 전체를 제공해줘.
예상 결과

엑셀을 업로드하면 요청 데이터가 표로 출력되고, 검색어를 입력하면 관련 요청만 조회된다.

다음 단계에서 이어질 내용

지역·업무분류·처리상태·우선순위·지연여부를 드롭다운으로 선택하는 필터 기능을 추가합니다.

다음 STEP 준비
  • 현재 HTML 저장 (STEP 1에서 생성한 HTML 코드 전체 보관)
  • 브라우저 실행 및 검색 동작 여부 자가 검증
  • 문제가 없으면 다음 프롬프트 복사 및 대기
💡 주의: STEP2부터는 예제 엑셀을 다시 첨부하지 않습니다. STEP1에서 생성한 HTML 코드를 계속 수정하여 이어서 점진적으로 기능을 확장해 나갑니다.
복구 프롬프트
방금 생성한 HTML이 정상 동작하지 않는다. 문제: - 엑셀 업로드 또는 표 출력 또는 검색 기능이 정상 동작하지 않는다. 요청: - HTML 파일 하나로 다시 만들어줘. - SheetJS CDN을 사용해서 엑셀을 읽어줘. - 업로드한 엑셀의 첫 번째 시트를 읽고 첫 번째 행을 컬럼명으로 사용해줘. - 전체 데이터를 표로 출력해줘. - 전체 컬럼 대상 통합 검색 기능을 넣어줘. - 브라우저 콘솔 오류가 나지 않도록 안정적으로 작성해줘. - 최종 HTML 코드 전체를 다시 출력해줘. 기준 파일: - 실습_표준_검색데이터.xlsx
02 STEP 02 / 조건 검색 기능 확장

고유값 자동 추출 드롭다운 필터 추가

목표: 자동 드롭다운 필터 추가 지역 / 업무분류 / 처리상태 필터 우선순위 / 지연여부 필터 초기화 버튼
설명

실제 개발 현장에서도 프로그램은 단 한 번에 만들어지지 않습니다. 처음에 간단한 초안을 완성한 뒤, 필요한 세부 기능을 하나씩 더해가며 고도화하는 것이 정석적인 방법입니다. 이번에는 기존에 타이핑만 하던 검색 방식을 획기적으로 개선해보겠습니다. 사용자가 직접 오타를 낼 걱정 없이, 마우스로 툭툭 누르기만 해도 엑셀 내부의 고유 데이터를 완벽하게 필터링할 수 있는 똑똑한 드롭다운을 결합할 예정입니다. 기존 결과물을 스스로 지속해서 업그레이드하고 가꾸는 참된 개발 프로세스를 이번 실습에서 몸소 체험해 보시기 바랍니다.

교육생 실습 내용
  1. STEP 1에서 만든 HTML 코드 전체를 LLM에 붙여넣는다.
  2. 아래 프롬프트를 입력한다.
  3. 새로 출력된 HTML 코드 전체로 기존 파일을 교체 저장한다.
  4. 브라우저에서 새로고침 후 엑셀을 다시 업로드한다.
입력할 프롬프트
아래 HTML 코드를 기반으로 기능을 확장해줘. 추가 기능: - 엑셀 업로드 후 실제 데이터의 고유값을 자동으로 추출해서 드롭다운 필터를 만든다. - 필터 대상 컬럼: 1. 지역 2. 업무분류 3. 처리상태 4. 우선순위 5. 지연여부 - 각 필터에는 "전체" 옵션을 넣는다. - 통합 검색과 드롭다운 필터가 동시에 적용되어야 한다. - 검색 결과 건수를 계속 표시한다. - 필터 초기화 버튼을 추가한다. - 기존 엑셀 업로드, 표 출력, 검색 기능은 유지한다. - HTML 파일 하나로 동작해야 한다. - 최종 HTML 코드 전체를 다시 출력해줘. [여기에 STEP 1에서 만든 HTML 코드 전체 붙여넣기]
예상 결과

지역, 업무분류, 처리상태, 우선순위, 지연여부를 선택하면 조건에 맞는 데이터만 조회된다.

다음 단계에서 이어질 내용

조회 결과를 관리자 관점에서 볼 수 있도록 요약 대시보드 지표를 구성합니다.

다음 STEP 준비

✓ 현재 HTML 파일을 저장한 후 다음 STEP을 진행하세요.

  • 드롭다운 목록이 업로드한 엑셀 값 기준으로 정상 자동화되는지 확인
  • 통합 검색 필터와 다중 드롭다운 필터의 교차 적용 테스트
  • 초기화 버튼 클릭 시 모든 드롭다운이 원상태로 돌아가는지 확인
복구 프롬프트
STEP 2에서 필터 기능이 정상 동작하지 않는다. 증상: - 드롭다운이 비어 있거나 - 필터 선택 후 결과가 나오지 않거나 - 검색과 필터가 함께 동작하지 않는다. 요청: - 기존 HTML 코드를 안정화해줘. - 엑셀 업로드 후 지역, 업무분류, 처리상태, 우선순위, 지연여부 컬럼의 고유값을 추출해서 드롭다운을 만들어줘. - 컬럼이 없을 경우 오류가 나지 않고 해당 필터만 건너뛰게 해줘. - 검색과 필터는 동시에 적용되게 해줘. - 필터 초기화 버튼도 정상 동작하게 해줘. - 기존 기능은 삭제하지 말고 전체 HTML 코드를 다시 출력해줘. [현재 문제가 있는 HTML 코드 전체 붙여넣기]
03 STEP 03 / 분석 시각화 요약

업무 현황 요약 대시보드 추가

목표: 대시보드 추가 전체 / 완료 / 진행중 건수 지연 / 우선순위 상 건수 완료율 자동 재계산
설명

단순히 엑셀 행을 줄지어 보여주는 것은 정보의 '나열'일 뿐, 진정한 '분석'이 아닙니다. 업무 관리자나 의사결정자가 이 데이터를 가치 있게 보려면 핵심 성과를 한눈에 관통하는 대시보드가 반드시 필요한데요. 이번 단계에서는 우리가 설정한 검색어와 드롭다운 선택 조건에 따라 실시간으로 대시보드 카드 지표가 바뀌는 놀라운 수식을 연동합니다. 수백, 수천 개의 복잡한 행 데이터 중에서 관리자나 실무자가 진짜 필요로 하는 통계를 마법처럼 뽑아내어 같은 데이터를 전혀 다른 거시적 관점으로 바라보고 유용하게 다루는 법을 이해해 보도록 하겠습니다.

교육생 실습 내용
  1. STEP 2에서 완성한 HTML 코드 전체를 LLM에 붙여넣는다.
  2. 아래 프롬프트를 입력한다.
  3. 출력된 HTML 코드로 기존 파일을 교체한다.
  4. 브라우저에서 엑셀 업로드 후 대시보드 숫자가 바뀌는지 확인한다.
입력할 프롬프트
아래 HTML 코드를 기반으로 대시보드 기능을 추가해줘. 추가 기능: - 현재 검색 및 필터 결과를 기준으로 대시보드를 계산한다. - 카드 형태로 아래 지표를 보여준다. 1. 전체 건수 2. 완료 건수 3. 진행중 건수 4. 대기 건수 5. 지연 건수 6. 우선순위 상 건수 7. 완료율 - 완료율은 완료 건수 / 전체 건수 기준으로 계산한다. - 처리상태 컬럼 값은 "완료", "진행중", "대기"를 기준으로 계산한다. - 지연여부 컬럼 값은 "지연"을 기준으로 계산한다. - 우선순위 컬럼 값은 "상"을 기준으로 계산한다. - 검색어나 필터가 바뀌면 대시보드도 즉시 다시 계산되어야 한다. - 기존 엑셀 업로드, 검색, 필터, 표 출력 기능은 유지한다. - HTML 파일 하나로 동작해야 한다. - 최종 HTML 코드 전체를 다시 출력해줘. [여기에 STEP 2에서 만든 HTML 코드 전체 붙여넣기]
예상 결과

필터 조건을 바꾸면 전체 건수, 완료율, 지연 건수, 우선순위 상 건수가 함께 변경된다.

다음 단계에서 이어질 내용

특정 행 클릭 시 세부 정보를 상세하게 조회할 수 있는 패널과 파일 다운로드 기능을 추가합니다.

다음 STEP 준비

✓ 현재 HTML 파일을 저장한 후 다음 STEP을 진행하세요.

  • 처리상태별 건수 집계가 엑셀 원본 실제 데이터 및 표 필터 데이터와 부합하는지 교차 확인
  • 지연여부 필터를 선택했을 때 대시보드의 실시간 지연 누계 상태 확인
  • 완료율 연산 지표가 최소 0%에서 최대 100% 한도 내로 오류 없이 출력되는지 확인
복구 프롬프트
STEP 3에서 대시보드 숫자가 맞지 않거나 필터 변경 시 갱신되지 않는다. 요청: - 현재 화면에 표시되는 필터링 결과 기준으로 대시보드를 다시 계산하게 수정해줘. - 원본 전체 데이터 기준이 아니라 현재 검색/필터 결과 기준이어야 한다. - 완료 건수는 처리상태가 "완료"인 행으로 계산해줘. - 진행중 건수는 처리상태가 "진행중"인 행으로 계산해줘. - 대기 건수는 처리상태가 "대기"인 행으로 계산해줘. - 지연 건수는 지연여부가 "지연"인 행으로 계산해줘. - 우선순위 상 건수는 우선순위가 "상"인 행으로 계산해줘. - 완료율은 완료 건수 / 현재 결과 전체 건수로 계산해줘. - 기존 기능은 유지하고 전체 HTML 코드를 다시 출력해줘. [현재 문제가 있는 HTML 코드 전체 붙여넣기]
04 STEP 04 / 상세 보기 및 결과 추출

상세보기 패널 및 CSV 다운로드 추가

설명

자주 묻는 질문

Q1 실무에서 다루는 엑셀 데이터는 행(Row)이 수만 건에 달합니다. 외부 서버나 DB 연동 없이 HTML 단일 파일(SheetJS CDN)만으로 대용량 데이터를 처리할 때 성능 저하나 브라우저 멈춤 현상은 없나요?

아주 날카로운 질문입니다. 본 커리큘럼에서 사용하는 방식은 브라우저의 클라이언트 자원(메모리 및 CPU)을 그대로 사용하는 100% 프론트엔드 동작 방식입니다. 일반적인 수천 건의 데이터는 지연 없이 실시간 필터링과 대시보드 연동이 가능하지만, 행이 수만 건 이상으로 넘어가면 브라우저가 일시적으로 멈추거나 렌더링 속도가 급격히 저하될 수 있습니다.

이를 해결하기 위해 STEP 5 이후의 심화 단계에서는 AI에게 "데이터를 한 번에 다 그리지 말고 50개씩 나눠서 보여주는 페이지네이션(Pagination) 기능을 추가해줘" 또는 "스크롤을 내릴 때마다 데이터를 로딩하는 무한 스크롤 방식을 적용해줘"라는 프롬프트를 통해 브라우저 과부하를 기술적으로 우회하고 최적화할 수 있습니다.

Q2 회사에서 다루는 데이터는 대외비나 개인정보가 포함된 민감한 보안 자료가 많습니다. 웹 브라우저에 엑셀을 업로드하고 LLM 소스코드를 활용하는 과정에서 내부 데이터가 외부로 유출될 우려가 전혀 없나요?

결론부터 말씀드리면, 모든 LLM 사용 시 개인정보 노출에 주의해야 합니다. 오늘 만든 로컬 기반 웹 앱은 오직 사용자의 PC(브라우저 메모리) 안에서만 읽고 분석하기 때문에 위험도가 낮습니다.

다만, 주의하실 점은 '코드를 생성하는 단계'입니다. STEP 5에서 나의 실제 업무 엑셀 구조로 커스텀하기 위해 AI에게 엑셀의 컬럼 구조를 알려주는 과정이 있습니다. 이때 실제 사내 데이터(이름, 주민번호, 내부 매출액 등) 내용물까지 통째로 LLM에 첨부하거나 프롬프트에 적어서는 절대 안 됩니다. 반드시 실제 데이터 행은 모두 지우고 '헤더(컬럼명)' 구조만 남긴 껍데기 파일이나 프롬프트 텍스트만 AI에게 제공하여 규칙을 학습시켜야 보안을 완벽하게 지킬 수 있습니다.

Q3 AI가 짜준 코드로 프로그램을 멋지게 완성했는데, 시간이 흐른 뒤 새로운 기능(예: 기간별 검색, 차트 시각화 등)을 또 추가하고 싶습니다. 제가 직접 코드를 수정할 줄 모르는데, 과거에 만들었던 이 단일 HTML 파일을 들고 AI와 어떻게 지속해서 유지보수 대화를 이어가야 하나요?

기초 과정에서 경험하셨듯, 바이브코딩의 핵심은 '이전 소스코드 전체를 AI에게 누적해서 먹여주는 것'입니다. 수개월이 지난 후라도 기존에 완성했던 업무요청검색_최종.html 파일을 메모장으로 열어 코드 전체를 복사합니다.

그리고 LLM 새 대화창에 "아래는 내가 과거에 완성해서 쓰던 업무용 HTML 프로그램이야. 이 코드를 바탕으로 [기간별 조건 검색 기능]을 추가하고 싶어. 기존 기능은 절대 삭제하거나 누락하지 말고, 전체 코드를 다시 출력해줘"라고 요청하시면 됩니다. AI는 과거의 맥락과 구조를 그대로 흡수하여 새로운 기능을 덧붙인 완벽한 업데이트 버전을 제공합니다.

Q4 HTML 단일 파일 방식은 간편하지만, 브라우저를 새로고침하거나 껐다 켜면 업로드했던 엑셀 데이터가 초기화되어 매번 다시 업로드해야 하는 번거로움이 있습니다. 데이터를 앱 내에 임시로라도 저장해둘 수 있는 방법은 없나요?

기초 단계의 단일 파일 프로그램은 휘발성 메모리를 쓰기 때문에 새로고침 시 초기화되는 것이 맞습니다. 하지만 여러분은 이미 AI를 파트너로 제어하는 법을 배웠으므로, 브라우저가 제공하는 내장 창고인 LocalStorage(로컬 스토리지) 기능을 AI에게 요구할 수 있습니다.

완성된 코드에 심화 프롬프트로 "사용자가 한 번 업로드한 엑셀 데이터를 브라우저의 LocalStorage에 암호화하여 저장해줘. 그래서 다음번에 브라우저를 새로고침하거나 다시 접속하더라도 자동으로 이전 데이터가 불러와져서 표와 대시보드에 바로 뜨도록 기능을 확장해줘"라고 지시해 보세요. 서버나 DB 없이도 마치 로그인해서 쓰듯 데이터를 유지해 주는 진정한 웹 애플리케이션으로 한 단계 진화시킬 수 있습니다.
